# Протоколы и маскировка

В системе PP используется протокол **pp-fallback**, предназначенный для имитации веб-трафика.

## pp-fallback

Данный протокол реализует механизм маскировки. Для внешних инструментов анализа трафика (DPI) соединение выглядит как стандартная сессия с веб-сайтом.

### Как это работает:
1.  **Маскировка под HTTP/2**: Весь служебный трафик прокси упаковывается в gRPC-потоки внутри стандартного HTTP/2 соединения.
2.  **Сайт-заглушка**: Если кто-то зайдет на ваш домен через обычный браузер, сервер покажет ему полноценный сайт. В зависимости от настроек это может быть:
    *   **Блог**: лента статей с комментариями и архивом.
3.  **Динамический контент**: Чтобы сайт не выглядел «мертвым», система автоматически наполняет его статьями. Она использует ключевые слова (**Теги**) для поиска и публикации тематического контента.

### Основные параметры протокола:

| Параметр | Описание |
| :--- | :--- |
| **Домен** | Ваш публичный домен, на котором будет работать сайт и прокси. |
| **Теги** | Ключевые слова (через запятую). По ним система будет искать контент для публикации в «Блоге» или «Форуме». |
| **gRPC Path** | Секретный путь (по умолчанию `/pp.v1.TunnelService/Connect`). Это «пароль» на уровне URL, который знают только клиенты. |
| **Noise Key** | Ключ для дополнительного слоя шифрования Noise, который защищает трафик внутри gRPC-туннеля. |
| **PSK** | (Pre-Shared Key) Основной ключ аутентификации пользователя. |

### Режимы Fallback (Тип сайта):
*   **Блог**: Идеально подходит для личного использования. Выглядит как современный минималистичный блог.
