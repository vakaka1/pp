# Инструкция по эксплуатации

В этом разделе описан процесс управления сервером через веб-интерфейс.

## Работа с подключениями

Раздел **Подключения** — это основное место настройки вашего сервера.

### Создание нового подключения
1.  Нажмите кнопку **Создать подключение**.
2.  Введите понятное **Название** (например, "Основной прокси").
3.  Укажите **Домен**, который вы привязали к серверу.
4.  В поле **Адрес** укажите локальный порт, например `8081`. 
5.  Настройте параметры маскировки (Тип сайта, Теги).

### Настройка HTTPS (SSL)
Чтобы прокси работал безопасно, ему нужен SSL-сертификат. PP автоматизирует этот процесс:
1.  В карточке подключения нажмите кнопку **Настроить**.
2.  Найдите раздел SSL и выберите режим **Let's Encrypt**.
3.  Система сама запустит `certbot`, получит сертификат и настроит Nginx для работы с вашим доменом.

### Управление Nginx
Нажав кнопку **Nginx** в карточке подключения, вы можете посмотреть автоматически сгенерированный конфиг для веб-сервера. PP использует Nginx как «фасад», который принимает трафик на 443 порту и передает его ядру.

## Управление клиентами

Клиенты — это пользователи или устройства, которым разрешен доступ к прокси.

1.  Нажмите кнопку **Клиенты** в карточке нужного подключения.
2.  В появившемся окне введите имя нового клиента и нажмите **Добавить**.
3.  В списке клиентов появятся кнопки:
    *   **Скачать** — загрузить файл `.json` для официального клиента.
    *   **Смотреть** — открыть окно с текстом конфига и **compact URI** (ссылка вида `ppf://...` для быстрого импорта).

## Обслуживание системы

В разделе **Ядро PP** вы можете следить за здоровьем сервера. Если в статусе написано «Конфигурация не совпадает» — нажмите **Обновить конфигурацию**, а затем **Перезапустить ядро**.

В разделе **О программе** можно проверить наличие обновлений и обновиться одной кнопкой, если вышла новая версия.
