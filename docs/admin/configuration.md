# Конфигурация системы

Система PP состоит из двух компонентов: **Ядра** (pp-core) и **Веб-панели** (pp-web). Панель управляет конфигурацией ядра, избавляя вас от ручного редактирования JSON-файлов.

## Как хранится конфигурация

Все настройки, которые вы вносите в интерфейсе, сохраняются в базе данных SQLite. На их основе панель генерирует итоговый файл для ядра:
`/var/lib/pp-web/generated/pp-core.json`

## Основные разделы конфигурации

### 1. Логирование
Вы можете настроить детализацию логов в разделе **Ядро PP**. Доступны уровни `debug`, `info`, `warn` и `error`. Логи помогают понять причину проблем с подключением.

### 2. Профили подключений
В разделе **Подключения** вы создаете «слушателей». Каждый профиль определяет:
*   **Протокол**: чаще всего это `pp-fallback`.
*   **Порт**: на каком локальном порту ядро будет ждать трафик (например, `127.0.0.1:8081`).
*   **Домен**: для какого домена выписывать SSL-сертификат.

### 3. Маршрутизация (Routing)
PP поддерживает централизованное управление правилами. Вы можете настроить правила в редакторе подключения:
*   **Типы правил**:
    *   `Сайты (geosite)` — списки доменов по категориям.
    *   `IP страны (geoip)` — например, направить трафик RU напрямую, а остальное через прокси.
    *   `Домен`, `IP/CIDR`, `Regexp` — для точечной настройки.
*   **Действия**:
    *   `прокси` — гнать через туннель.
    *   `напрямую` — идти в интернет с IP сервера.
    *   `блокировать` — обрывать соединение.

## Синхронизация

После изменения любых настроек в интерфейсе необходимо зайти в раздел **Ядро PP** и выполнить:
1.  **Обновить конфигурацию**: панель перезапишет файл `pp-core.json`.
2.  **Перезапустить ядро**: служба `pp-core` применит новые настройки.
