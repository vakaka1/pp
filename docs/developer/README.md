# Документация для разработчиков

Этот раздел предназначен для разработчиков, которые хотят создавать собственные GUI-клиенты или автоматизировать работу с прокси-системой PP.

Основным инструментом для интеграции является бинарный файл **pp-client**. Он спроектирован так, чтобы его можно было легко использовать как «движок» (backend) для графических оболочек на любой платформе.

## Содержание

1.  **[pp-client как движок](engine.md)** — основные команды, принципы работы и взаимодействие с GUI.
2.  **[Конфигурация клиента](config-client.md)** — детальное описание структуры JSON-файла для настройки подключения.
3.  **[Гайд по интеграции](integration.md)** — как читать логи, обрабатывать ошибки и управлять системным прокси (Full Tunnel).
