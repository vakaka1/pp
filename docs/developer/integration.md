# Гайд по интеграции с GUI

Этот раздел содержит практические рекомендации по встраиванию `pp-client` в ваше приложение.

## Установка и обновление pp-client
Для инициализации и обновления уже настроен скрипт:
```bash
curl -fsSL https://raw.githubusercontent.com/vakaka1/pp/main/scripts/install-client.sh | bash
```
На сервере уже настроен CI/CD с помощью которого вы всегда сможете проверять и получать актуальную версию `pp-client`.

## Управление процессом

Мы рекомендуем запускать `pp-client` как дочерний процесс.

**Важные моменты:**
1.  **Завершение**: При закрытии GUI обязательно отправляйте процессу сигнал `SIGTERM` или `SIGINT`. Клиент корректно закроет все соединения и очистит правила маршрутизации (если был включен Full Tunnel).
2.  **Перезапуск**: Если пользователь меняет сервер или порт, лучше завершить текущий процесс и запустить новый с новым конфигом.

## Мониторинг состояния (Логи)

Клиент пишет логи в `stdout`. Для GUI важно отслеживать ключевые сообщения:

*   **Успешный запуск**: 
    `info: SOCKS5 server started {"address": "127.0.0.1:1080"}` — это сигнал, что движок готов принимать трафик.
*   **Ошибки подключения**:
    `error: failed to open stream` — сервер недоступен или ключи неверны.
*   **Маршрутизация**:
    В режиме `--verbose` клиент пишет, какое правило применилось к каждому домену. Это можно использовать для отладки в «режиме разработчика» вашего GUI.

## Работа с Full Tunnel (Linux)

Режим полнопоточного туннеля требует прав суперпользователя, так как он изменяет правила `iptables`.

**Рекомендация для GUI:**
Для выполнения команд `up` и `down` используйте `pkexec` или `sudo`. 
Пример:
```bash
pkexec pp-client full-tunnel up --config client.json
```
Убедитесь, что вы вызываете `down` перед выходом из приложения, иначе у пользователя может пропасть интернет (трафик будет пытаться уйти в несуществующий прокси).

## Совет по безопасности
Не храните PSK и приватные ключи в открытом виде в логах GUI. Хотя `pp-client` скрывает чувствительные данные в обычном режиме, при уровне `debug` некоторые параметры могут попасть в вывод.
