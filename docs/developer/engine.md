# pp-client как движок прокси

Бинарный файл `pp-client` — это сердце клиентской части системы. Он берет на себя всю сложную работу: установку gRPC-соединения, Noise-шифрование, маскировку трафика и локальную маршрутизацию.

## Основная концепция

Для создания GUI-клиента вам не нужно реализовывать протоколы PP внутри своего кода. Ваше приложение (на Electron, Qt, Flutter или Go) выступает в роли «оболочки», которая:
1.  Формирует JSON-файл конфигурации.
2.  Запускает `pp-client` как дочерний процесс.
3.  Следит за состоянием туннеля через вывод логов в `stdout`.

## Команды управления

### Запуск прокси
Основная команда для работы:
```bash
pp-client start --config client.json
```
При запуске клиент открывает локальные порты (SOCKS5/HTTP), указанные в конфиге, и начинает пересылать трафик на сервер.

### Проверка конфигурации
Полезно для валидации данных перед запуском:
```bash
pp-client validate-config --config client.json
```

### Системный прокси (Linux Full Tunnel)
Если ваш GUI должен перенаправлять **весь** трафик системы (а не только тех приложений, где настроен прокси), используйте скрытую команду:
```bash
# Включить (требуются права root для работы с iptables)
sudo pp-client full-tunnel up --config client.json

# Выключить
sudo pp-client full-tunnel down
```

## Управление профилями и URI 🔗

Одной из самых удобных возможностей для разработчика GUI является встроенная система управления профилями. Она позволяет избежать ручного создания JSON-файлов и упрощает хранение настроек.

### Импорт из URI
Вы можете импортировать настройки напрямую из ссылки формата `ppf://`:
```bash
pp-client import "ppf://user@domain.com:443?pub=KEY&psk=KEY&path=/grpc"
```
Эта команда распарсит URI и создаст готовый JSON-конфиг в системной папке или рядом с бинарником.

### Работа со списком профилей
Чтобы получить список всех импортированных серверов для отрисовки в GUI, используйте:
```bash
pp-client list --json
```
Флаг `--json` вернет массив объектов с именами, путями к файлам и метаданными. Это позволяет вашему приложению мгновенно синхронизировать интерфейс с базой профилей.

### Запуск по имени
После импорта вам не нужно указывать путь к файлу. Достаточно использовать имя профиля:
```bash
pp-client start [имя_профиля]
```

### Удаление профиля
```bash
pp-client delete [имя_профиля]
```

## Как это работает внутри
*   **Места поиска**: Клиент ищет конфиги в `/etc/pp/`, в локальной папке `configs/` или в текущем каталоге.
*   **Слушатели**: Клиент одновременно может поднять SOCKS5, HTTP и Прозрачный (Transparent) слушатели.
*   **Пул соединений**: Клиент поддерживает постоянное соединение с сервером, что минимизирует задержки (Latency) при открытии новых сайтов.
*   **Маршрутизация**: Внутри клиента работает движок, который решает, какие сайты пускать через прокси, а какие — напрямую.
